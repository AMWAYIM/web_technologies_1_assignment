How to set-up a virtual environment using venv

Documentation 1

1

Create a folder with the name of the project.

$ mkdir LEARN-MYSQL

2

Initial installation of the venv module:

sudo apt install python3-venv

Create the virtual environment within the project folder.

$ cd LEARN-MYSQL
# Create a virtual environment called venv
$ python3 -m venv venv
# pyenv.cfg filecontains details about the installed python version

3

Syntax

# Method-1
source NAME_OF_ENVIRONMENT/bin/activate
# Method-2
. NAME_OF_ENVIRONMENT/bin/activate

How to activate a virtual environment

source venv/bin/activate

Or

. venv/bin/activate

After activating the virtual environment no need to explicitly call pip 3 or python 3 as the virtual environment was created using a version of python 3.

Both python & pip link to the same executable files as python 3 & pip 3.

4

Update pip:

python -m pip install --upgrade pip
(venv) $ python -m pip install --upgrade pip

To update pip automatically when creating a virtual environment:

$ python3 -m venv venv --upgrade-deps

To avoid installing pip in a virtual environment:

$ python3 -m venv venv --without-pip

To print a list of installed packages within the virtual environment:

$ pip freeze

To write the list of installed packages to a file:

$ pip freeze > requirements.txt

5

Install required packages:

# Installing individual packages
(venv) $ pip install mysql-connector-python --only-binary=:all:
# Installing multiple packages from a requirements.txt file
(venv) $ pip3 install -r requirements.txt --only-binary=:all:

Installing from a requirements file:

pip install -r requirements.txt --only-binary=:all:

To use external packages installed in the base python version within the virtual environment:

$ python3 -m venv environment2 --system-site-packages

6

To overwrite an already existing virtual environment:

$ python3 -m venv venv --clear

7

Use help for more information about a command & its options:

pip help [command]

example:

pip help install
